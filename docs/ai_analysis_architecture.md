# AI Analysis Architecture

## AI Guidelines

AIは以下の方針で評価を行う。
- 数値だけではなく案件全体の文脈を考慮する
- 判断理由を必ず説明する
- 不確実な内容は推測ではなく「可能性」として表現する
- 明らかな根拠がない内容は評価に含めない
- ユーザーが応募判断できる情報を優先して提示する
- 必ず長所と短所の両方を提示する

## Goal
ユーザーに最適な案件を多面的に評価し、応募優先度の高い順に提示するとともに、応募判断に必要な根拠と戦略を提供する。

## Input
### Jobクラスより案件情報を参照
- title: 案件タイトル
- description: 案件詳細
- reward: 報酬情報
- application_deadline: 応募締め切り
- published_at: 応募開始日
- application_count: 応募人数
- recruitment_count: 募集人数

### UserProfileクラスよりユーザーの個人情報を参照 (将来的に実装)
skills: list[str]
experience: list[str]
interests: list[str]
preferred_work_style: list[str]
goals: list[str]
※ UserProfileは今後拡張予定

## Output

### 数値評価
- 報酬: int (1〜5)
- 競争率: int (1〜5)
- AI適性: int (1〜5)
- 継続案件: int (1〜5)
- 募集内容: int (1〜5)
- あなたとの適合度: int (1〜5)
- 総合評価: int (0〜100)

### おすすめな理由
list[str]  
以下例
- あなたのUnity・AI開発経験と親和性が高い
- 生成AIを業務で活用する内容が中心
- 継続案件で安定収入につながる可能性がある

### 注意点
list[str]  
以下例
- 応募人数がやや多く競争率は中程度
- ポートフォリオの質が採用可否に影響しそう

### 応募戦略
list[str]  
以下例
- Unityでの開発実績を前面に出す
- 生成AIを活用したプロジェクト経験を具体的に記載する
- AIを使った課題解決力をアピールする

## Evaluation Criteria
AIは以下の観点から案件を評価し、それぞれ5段階評価を行う。

### 1. 報酬
評価観点
- 報酬金額
- 時給換算の妥当性
- 成果報酬か固定報酬か
- 作業内容に対するコストパフォーマンス

### 2. 競争率
評価観点
- 応募人数
- 募集人数
- 掲載日からの経過日数
- 人気案件である可能性

### 3. AI適性
評価観点
- ChatGPTなど生成AIの活用が許可・推奨されているか
- AIによる効率化が期待できるか
- AIスキルを活かせる案件か

### 4. 継続案件
評価観点
- 長期契約の可能性
- 継続依頼の記載
- 定期的な業務であるか

### 5. 案件品質
評価観点
- 業務内容が具体的か
- 求めるスキルが明確か
- 成果物が定義されているか
- 発注者の意図が読み取りやすいか

### 6. あなたとの適合度
評価観点
- 保有スキルとの一致度
- 過去の経験との一致度
- 興味・得意分野との一致度
- 希望する働き方との一致度
- 将来の目標との一致度

### 総合評価
総合評価は上記の評価項目を総合的に判断し、100点満点で算出する。  
点数の単純平均ではなく、案件全体の魅力度や将来性、応募価値も考慮して評価する。  
AIは評価結果だけではなく、
- 評価理由
- 注意点
- 応募時の戦略
についても説明する。

#### Weight
AIは以下を参考に総合評価を行う。
- 報酬：20%
- 競争率：15%
- AI適性：20%
- 継続案件：15%
- 案件品質：20%
- あなたとの適合度：10%
